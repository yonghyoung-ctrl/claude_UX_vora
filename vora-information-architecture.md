# VORA · 정보구조 & UX 원칙 문서

> 제품 컨셉 정리부터 화면별 UX 가이드라인까지, 팀 공유용 레퍼런스.
> 작성: 2026년 4월 · 대상: 디자인·기획·엔지니어링

---

## 1. 제품의 본질

VORA는 단순한 AI 챗봇이 아니라 **"페르소나 AI들이 함께 살아가는 소셜 네트워크"**입니다. 이 정체성은 세 가지 구조가 맞물려 만들어집니다.

### 1.1 세 가지 AI 레이어

| 레이어 | 정체 | 누가 만드는가 | 누가 쓰는가 |
|---|---|---|---|
| **나의 AI** | 나의 페르소나를 학습한 AI | 내가 직접 학습시킴 | 나 + 내가 허락한 친구들 |
| **친구 AI** | 친구의 페르소나를 학습한 AI | 친구가 직접 학습시킴 | 친구가 부재중이어도 내가 호출 가능 |
| **VORA AI** | 서비스가 제공하는 실용 에이전트 | VORA 팀 (도구 조합) | 모두. PPT 제작·음악 생성·이메일 예약 등 실용 목적 |

**핵심 의미**: 나의 AI는 "나 대신" 친구에게 답할 수 있는 사회적 존재입니다. 이것이 VORA를 다른 AI 서비스와 구분 짓는 본질적 차이점입니다.

### 1.2 세 가지 대화 모드

모드는 "같은 채팅 UI의 변형"이 아니라 **사실상 별개의 워크플로우**입니다.

| 모드 | 기본 구성 | 성격 | UX 패턴 |
|---|---|---|---|
| **채팅** (Chat) | 1:1 (나 ↔ AI 1명) | 즉시성 · 가벼운 대화 | 전통적 메신저 UI |
| **디스커션** (Discussion) | 멀티 에이전트 (여러 AI가 토론) | 관전형 · 의견 대립 유도 | 대본·스레드 UI |
| **파워리서치** (Power Research) | PDCA 기반 단계형 | 비동기 프로젝트 | 칸반·타임라인 UI |

어느 모드든 **친구 AI를 초대**할 수 있고, 디스커션과 파워리서치는 **단계별 담당 AI 교체**가 가능합니다.

### 1.3 소셜 루프

```
내가 내 AI를 학습시킴
  └→ Level up (참고 정보 바이트 수 기준)
       └→ 친구가 내 AI를 호출해 대화
            └→ "내 AI가 N번 쓰였어요" 피드백
                 └→ 더 학습시키고 싶은 동기
```

이 루프가 돌아가게 하려면 **"친구가 내 AI를 얼마나 썼는지"가 가장 눈에 띄는 지표**가 되어야 합니다.

## 2. UX 위계 원칙 (Design Rules)

모든 화면 설계에 공통 적용되는 5가지 규칙.

### 원칙 ①  One Hero Per Screen

각 화면마다 **주인공(Hero) 1개**를 정하고, 나머지는 조연(Supporting)·배경(Ambient)으로 처리합니다.

- Hero는 화면 면적의 **55~65%**를 차지해야 함
- Supporting 20~30%, Ambient 10~15%
- 지금 VORA는 대략 33/33/33 — 평면적인 원인

| 화면 | Hero | Supporting | Ambient |
|---|---|---|---|
| 홈 | 입력창 + 모드 선택 + AI 초대 | 친구 AI · 소셜 피드 | 레벨 · VORA AI 안내 |
| 채팅 | 메시지 영역 + 입력창 | 관련 질문(다음 행동) | 채팅 리스트(접이식) |
| 파워리서치 | 단계 카드 보드 | 단계별 AI 할당 | 스테이지 트래커 |
| 프로필 | AI 성장 카드 + 업데이트 CTA | AI 베이스 3요소 | 도구 · 친구 기여도 |

### 원칙 ②  컬러 예산제

보라색은 "행동"에만, 회색 계열은 "상태"와 "라벨"에만.

- **Solid 보라** = 실행 가능한 **주요 행동** (전송, 업데이트, 시작) — 한 화면에 1~2개만
- **보라 아웃라인/소프트** = 선택됨 · 활성 상태
- **회색 tonal** = 메타데이터, 라벨, 필터, 비활성

> 현재 VORA는 레벨 배지, 탭 활성, 버튼, 토글 등 모든 UI가 보라를 쓰고 있어서 **보라가 "기본"이 되어버렸습니다.**

### 원칙 ③  크기 비율 1.6배 룰

Primary 요소는 Secondary보다 **최소 1.6배** 크게(높이·폰트·영역). 지금은 1~1.1배라 위계가 생기지 않습니다.

### 원칙 ④  리스트는 3층 구조

모든 리스트(채팅·친구 AI·도구·지식창고)에 공통 적용:

- **Section 헤더**: 굵게, 대문자/라벨, 액션(+추가, 전체 보기)
- **Item 본문**: 16px, 1줄 제목 + 1줄 요약
- **Meta**: 12px, 회색, 시간·개수·아이콘

### 원칙 ⑤  모든 화면에 컨텍스트 바

"배너 영역입니다" 같은 정체성 없는 바를 제거하고, 화면별 맥락 바를 둡니다:

- 홈: "👋 형용준님, 좋은 오후예요"
- 프로필: "형용준 AI 관리 · Lv.5"
- 채팅: "김루하 AI · 채팅 모드 · 이미지 생성 도구 활성"
- 파워리서치: "🔍 파워리서치 · 경쟁사 BM 분석 · 4단계 · 예상 12분"

## 3. 정보구조 (Information Architecture)

### 3.1 내비게이션 계층

```
Root
├─ 홈                     ← AI 시작점 + 소셜 피드
├─ 채팅                   ← 모든 대화 기록 (채팅/디스커션/파워리서치 통합 리스트)
├─ 도구 상점               ← VORA AI + 유료 도구 구독 관리
├─ 보관함 (지식 창고)       ← 업로드한 문서 · AI 생성 파일
├─ 프로필 (내 AI 관리)     ← 레벨, 베이스 편집, 친구 기여도
├─ 친구 초대               ← 공유 링크, QR
└─ 설정                   ← 계정, 알림, 공유 범위
```

### 3.2 화면별 JTBD (Jobs to Be Done)

사용자가 각 화면에서 **해결하려는 진짜 문제**.

| 화면 | 사용자의 JTBD | 지원해야 하는 주요 액션 |
|---|---|---|
| 홈 | "뭔가 AI와 작업을 시작하고 싶다" | 입력·모드 선택·친구 초대·이어하던 대화 복귀 |
| 채팅 리스트 | "이전에 했던 그 대화 찾기" | 검색, 고정, 모드별 필터, 북마크 |
| 채팅 대화창 | "AI와 실시간으로 문제 풀기" | 메시지 송수신, 다음 행동 제안, 근거 확인 |
| 파워리서치 Plan | "조사 계획이 타당한지 보고 수정하기" | 단계 편집, AI 할당, 예상 시간 파악 |
| 파워리서치 Do/Check | "AI들이 제대로 일하고 있는지 지켜보기" | 실행 상태 확인, 일시정지, 재실행 |
| 프로필 | "내 AI를 더 똑똑하게 만들기" | 자료 추가, 대화 반영, 레벨 확인 |
| 도구 상점 | "내 AI에 새 능력 부여하기" | 도구 탐색, 구독, 친구와 공유 |

### 3.3 상태 전환

주요 사용자 여정의 상태 변화:

```
가입 (Lv.0)
  ↓ 이름·사진·참고할 정보 입력
활성화 (Lv.1+)
  ↓ 지식창고 업로드 → 자료 기반 업데이트
베이스 확장 (Lv.2~4)
  ↓ 친구 초대 → 친구 AI 추가
소셜 연결 (Lv.3+)
  ↓ 친구가 내 AI 호출 시작
피드백 루프 (Lv.4+)
  ↓ 채팅 기반 업데이트 반복
파트너 (Lv.5)
  ↓ 2MB 베이스 달성
```

## 4. 화면별 가이드라인

### 4.1 홈

**Hero**: 입력창 + 모드 탭(내부) + 친구 초대 칩

**핵심 변화**
- 기존 상단 탭(채팅/파워리서치/디스커션) → **입력창 내부 모드 선택**으로 통합
- 배너 영역 → 맥락 바로 교체
- "VORA AI 준비 중" 빈 박스 → **실용 에이전트 미리보기 스트립**
- "오늘 하이라이트" 피드로 소셜 루프 가시화

**포함해야 할 요소**
1. 맥락 바 (인사말 + 검색·알림)
2. Hero 입력창 (모드 · 프리셋 · 친구 초대 통합)
3. 소셜 피드 ("오늘 내 AI가 불려간 횟수")
4. 친구 AI 카드 그리드 (활동 상태 표시)
5. VORA 실용 에이전트 스트립
6. 내 AI 성장 카드 (프로필 진입점)

### 4.2 채팅 (대화창)

**Hero**: 메시지 영역 + 입력창 (전체 60%+)

**핵심 변화**
- **대화 헤더 신설** — 누구·어떤 모드·어떤 도구가 활성인지 항상 표시
- 채팅 리스트는 **접이식 사이드**(280px, 접을 수 있음)
- "관련 질문" → "다음으로 해볼 만한 것"으로 리네이밍 + **액션 칩 스타일**(soft fill + 화살표)
- **도구 태그 vs 액션 칩 시각 분리**: 도구 태그 = outline, 액션 칩 = fill
- FAST/DEEP와 지식창고 토글을 **입력창 내부로 통합**

### 4.3 채팅 (리스트)

**3층 위계**
- **📌 고정 섹션** (최상단)
- **오늘 / 어제 / 이번 주 / 더 과거** (날짜 그룹)
- 안 읽은 아이템 = **보라 점 + 굵은 텍스트**
- 모드별 컬러 배지: 채팅(보라) · 파워리서치(파랑) · 디스커션(주황)

### 4.4 파워리서치 (PDCA 보드)

**Hero**: Plan 단계의 단계 카드 보드

**핵심 원칙**
- 채팅 UI가 아닌 **프로젝트 보드 UI**로 완전히 다른 제품처럼 취급
- 4단계 스테이지 트래커를 상단에 명시 (Plan → Do → Check → Act)
- 각 단계 카드에 **AI 할당 칩** (클릭 시 팝오버로 친구 AI 교체)
- **인플레이스 편집** (제목, 설명, 순서 변경)
- 실행은 단일 CTA `🚀 이 계획으로 시작`

**Do 단계 원칙**
- 실시간 실행 상태 (완료 ✓ / 진행 중 ⟳ / 대기)
- 스트리밍 출력을 인라인으로 표시
- 언제든 일시정지 가능

**Check 단계 원칙**
- 단계별 결과 카드
- 각 카드에 `👍 만족` / `🔄 다시 실행` / `🤖 다른 AI로 재실행` / `✏️ 수정 요청`

**Act 단계 원칙**
- 종합 리포트 + "더 깊이 파고들기" 후속 액션 칩
- 결과물 내보내기 (PDF · 공유 · 북마크)

### 4.5 프로필

**Hero**: "내 AI 성장시키기" 카드

**핵심 변화**
- 큰 레벨 카드 → 이름 옆 `⭐ Lv.5` 배지로 압축
- **Level 진행 바** + 구체 목표 ("800KB 더 알려주면 Lv.6")
- 자료/채팅 업데이트 2개 CTA를 나란히, 각 설명·마지막 업데이트 시점 표시
- **AI 베이스 3요소** (참고할 정보 / 지식창고 / 한 줄 소개)를 병렬 배치, 공개·비공개 뱃지로 역할 구분
- "내 AI가 친구에게 기여한 정도" 섹션 (숫자 + 친구 아바타)

### 4.6 도구 상점

**리프레이밍**: "도구"가 아닌 **"내 AI의 능력"** 또는 **"가능한 작업"**

**이유**
- 사용자가 도구를 일일이 호출하지 않음 — AI가 자동 호출
- 따라서 "쇼핑"보다 **"능력 부여"** 메타포가 적합
- 구독 중 / 무료 / 유료 구분 명확히

### 4.7 보관함 (지식창고)

**3층 위계 적용**
- 파일 카드 (썸네일 · 이름 · 크기 · 분석 상태)
- 메타: 기본 분석 · 심화 분석 · 키워드 그래프 상태
- 용량 바 (11.5MB / 100MB) — 프로필 레벨과 연동

## 5. 핵심 UX 결정 사항

### 5.1 RAG UX → 옵션 C (하이브리드) 채택

FAST/DEEP 토글과 지식창고 검색 토글을 **자동 판단 + 뱃지 + 근거 표시**로 재설계.

```
입력창:  [🎯 기본] 메시지 입력... [📎 🎤 → 전송]
           ↑ 탭하면 프리셋 시트 (빠른 / 기본 / 정확)

응답 하단: 🎯 기본 · 📄 내 자료 2개 참조 (file1.pdf, file2.pdf)
```

**단계적 출시 전략**
- **1차**: 옵션 B(완전 자동 + 근거 표시)로 릴리스. 자동 분류 정확도를 실데이터로 튜닝
- **2차**: 뱃지 UI 추가해 옵션 C로 완성

### 5.2 멀티 에이전트 UI

친구 AI 초대는 **모든 모드에서 입력창 내 1차 액션**으로 승격.

- 채팅: `+ 친구 AI 초대` 칩 (입력창 내부)
- 디스커션: 기본값이 멀티 — 상단에 참여자 칩 나열
- 파워리서치: 단계별 담당자 칩 (클릭 시 스왑)

AI 간 턴 관리는 AI들이 자율적으로, 사용자는 직접 입력만.

### 5.3 레벨 시스템의 프레이밍

레벨을 **"유지보수 상태"가 아닌 "성장 게임"**으로.

- "Lv.5 · 완벽한 파트너에요!" (추상적) → **"Lv.5 · 1.2/2.0 MB · 800KB 더 알려주면 Lv.6"** (구체적)
- Level up 시 축하 애니메이션 + 가능해진 새 능력 안내
- 프로필 진입 시 이 카드가 Hero

### 5.4 소셜 시그널의 가시성

"내 친구가 사용 (13)"이 제품의 핵심 가치 지표.

- 홈 화면 "오늘 하이라이트" 피드로 상승
- 프로필에 "이번 주 내 AI가 친구 8명에게 13번 쓰였어요" 섹션
- 알림: 친구가 내 AI를 처음 부른 순간에 푸시

## 6. 우선순위 로드맵

### Phase 1 — 즉시 체감되는 변화 (1~2주)

**Quick Wins. 구조 변경 없이 시각적 위계만 바로잡기.**

- [ ] 배너 영역 제거 → 화면별 컨텍스트 바 도입
- [ ] 입력창 크기·그림자 업그레이드 (모든 화면 공통 컴포넌트화)
- [ ] 채팅 대화 헤더 신설
- [ ] "관련 질문" → "다음으로 해볼 만한 것" 리네이밍 + 액션 칩 스타일 적용
- [ ] 보라색 사용 오디트 — 한 화면당 Solid 보라 1~2개 원칙 강제

### Phase 2 — 구조 재편 (3~4주)

**정보구조 변경이 필요한 작업.**

- [ ] 홈의 탭(채팅/파워리서치/디스커션)을 입력창 내 모드 칩으로 이동
- [ ] 프로필 재설계 — "성장 카드"를 Hero로
- [ ] 채팅 리스트 3층 위계 적용 (고정 · 날짜 · 모드 배지)
- [ ] 파워리서치 PDCA 보드 UI로 전환 (채팅 UI 탈피)
- [ ] "내 친구가 사용" 피드를 홈 화면 전면 배치

### Phase 3 — 시스템화 (5~8주)

**장기적 일관성 확보.**

- [ ] 디자인 토큰 정리 (Figma 시스템)
- [ ] RAG UX → 옵션 B 릴리스 후 옵션 C 점진 이행
- [ ] 도구 상점 리프레이밍 ("도구" → "능력")
- [ ] 멀티 에이전트 UI 패턴 공통 컴포넌트화
- [ ] 레벨 게이미피케이션 (축하 애니메이션, 신규 능력 안내)
- [ ] 빈 상태 가이드라인 전면 정리

## 7. 열린 질문 / 추가 논의 필요

팀에서 결정·합의가 필요한 포인트.

1. **VORA AI의 위상** — 친구 AI와 동급으로 대우할 것인가, 별도 카테고리(도구)로 둘 것인가?
2. **친구 AI 호출 시 누적 데이터 정책** — 친구가 내 AI를 부른 대화는 내가 볼 수 없지만, AI 학습에는 반영되나?
3. **디스커션에 내 AI 투입 시 정체성** — 내 AI가 내 의견을 대변하는가, 독립적 에이전트인가?
4. **파워리서치 결과의 소유권** — 여러 AI가 협업한 리포트는 누구의 저작물?
5. **도구 공유 기본값** — "친구 AI에게 내 유료 도구 허용"을 기본 ON으로 할지 OFF로 할지? (현재 ON)
6. **레벨 시스템 리워드** — 레벨업으로 얻는 실질적 혜택은? 기능 해제? UI 커스터마이징? 상징적?
7. **30일 자동 삭제** — 파워리서치 결과도 30일이면 아쉬움. 북마크 UX를 더 잘 보이게 하거나 정책 차별화 필요.
