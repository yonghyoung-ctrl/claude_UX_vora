RAG UX · 3가지 옵션 비교

현재 VORA는 FAST/DEEP 토글과 내 지식창고 검색 토글을 우하단 플로팅 패널에 띄워 RAG의 존재를 강조하고 있어요. 하지만 RAG 개념 자체가 생소한 사용자에게는 오히려 "뭘 골라야 하지?" 하는 결정 피로를 유발합니다. 세 가지 대안을 실제로 조작해보며 비교해보세요.

💡
설계 원칙: "RAG를 설명한다"가 아니라 "고르지 않아도 되게 한다". 강조가 필요하다면 답변 자체에 근거를 붙이는 방식이 토글보다 훨씬 강력해요.

OPTION A 프리셋 3종 · 상황형 라벨

FAST/DEEP와 지식창고 ON/OFF를 4가지 조합 중 3개 프리셋으로 통합. 기술 용어 대신 사용 상황으로 라벨링.

✓ 장점

  • "FAST/DEEP/RAG" 같은 기술 용어가 사라짐 — 사용 상황으로 이해
  • 선택지가 4개→3개로 줄어 인지 부담 감소
  • 프리셋이라 조합 오류 없음 (DEEP+지식창고 OFF 같은 애매한 조합)

✕ 단점

  • 여전히 매번 선택해야 함 — 기본값을 "균형"으로 숨기면 결국 변경을 안 함
  • 왜 이 답변이 나왔는지 사후에 알 수 없음 (근거 부재)
  • 입력창 상단이 pill로 차 있어 "먼저 고르고 적어라"는 압박

OPTION B 자동 + 사후 근거 표시

사용자는 아무것도 선택하지 않음. AI가 질문 유형에 따라 자동으로 결정하고, 답변 하단에 "어떻게 답했는지"를 표시.

✓ 장점

  • 입력 단계에서 아무것도 선택할 필요 없음 — 가장 낮은 진입 장벽
  • 근거 표시로 신뢰 신호 생성 ("진짜 내 자료를 봤구나")
  • "다르게 답해줘" 버튼으로 불만족 시에만 가벼운 재선택
  • RAG를 자연스럽게 학습 — 근거 칩을 보며 점진적으로 개념 체득

✕ 단점

  • AI의 자동 판단이 틀리면 사용자는 "왜 이렇게 답했지?"에서 시작해야 함 (재시도 비용)
  • 속도/비용 우선인 사용자가 매번 DEEP으로 답변 받으면 불만
  • "내가 통제하고 싶다"는 파워 유저에게는 답답할 수 있음

OPTION C ★ 추천 하이브리드 · 자동 기본 + 입력창 내 뱃지 + 근거

B의 자동화 + A의 선택권 + 근거 표시를 모두 결합. 평소엔 보이지 않지만, 원할 때 뱃지를 탭해 즉시 변경.

✓ 장점

  • B의 자동화 장점 + A의 명시적 제어권을 모두 확보
  • 뱃지가 현재 상태를 1초에 알려줌 — 입력 전에 슬쩍 보고 안심 또는 변경
  • 근거 칩이 답변 하단에 항상 있어서 RAG를 자연스럽게 학습
  • 플로팅 패널이 사라지면서 우하단 공간 확보 — 더 깔끔한 UI

✕ 단점

  • 구현 복잡도가 A·B보다 살짝 높음 (자동 판단 + 수동 오버라이드 + 근거 표시)
  • 뱃지 인식이 안 되면 사용자는 B처럼 사용 (하지만 이건 단점이 아니라 안전망)

★ FINAL RECOMMENDATION

옵션 C · 하이브리드 방식으로 가는 것을 추천

이유: VORA의 RAG 강조 의도(= 사용자가 RAG를 인지하길 바람)는 토글이 아니라 근거 칩으로 훨씬 잘 해결됩니다. "📄 내 자료 2개 참조"라는 사후 표시가 "지식창고 ON 토글"보다 훨씬 강력한 신뢰 신호예요. 동시에 제어권이 필요한 순간엔 입력창 뱃지 한 번의 탭으로 변경 가능하니, 초보자와 파워 유저 모두 만족시킬 수 있어요.

단계적 출시 전략: 먼저 옵션 B(자동 + 근거)를 1차 릴리스로 내보내 자동 분류 정확도를 실데이터로 튜닝하고, 그 다음 뱃지 UI를 추가해 옵션 C로 완성하는 경로가 리스크가 가장 낮아요.
